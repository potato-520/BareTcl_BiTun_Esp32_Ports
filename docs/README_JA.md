# BareTcl & BiTun ESP32 移植・統合プロジェクト

本项目は、超軽量スクリプトインタプリタである **BareTcl** と、セキュアなネットワークプロキシトンネルである **BiTun** を、ESP-IDFフレームワークを用いてESP32プラットフォームに移植し、統合したネイティブC言語プロジェクトです。

本プロジェクトにより、ESP32チップ上で動作する超軽量の対話型Tclコマンドラインシェルが提供されます。GPIOなどのハードウェア制御、ネットワーク診断（Ping/Ipconfig）、不揮発性システム設定の保存（NVS）、およびセキュアなBiTun SOCKS5プロキシトンネル（KCPマルチプレクシングおよびChaCha20-Poly1305 AEAD暗号化）の動的制御などが実現します。

---

## 1. 主な機能 (Key Features)

*   **軽量Tclエンジン (BareTcl)**:
    *   外部の動的メモリ割り当て（`malloc`/`free`）への依存は一切ありません。実行環境は固定された静的メモリプール（Arena）内に完全にバインドされているため、リソース制限の厳しいマイコン環境でも極めて堅牢に動作します。
    *   ANSIカラーコードおよび非ブロッキングな文字単位の入力に対応した、インタラクティブなコンソールシェルを提供します。
    *   GPIO制御（ピンモード設定、デジタル入出力）、スリープディレイ（`sleep`）、ログ詳細度制御（`log`）、ネットワーク状態表示（`ipconfig`）、およびICMP診断（`ping`）などのカスタムコマンドを登録しています。
*   **不揮発性Key-Valueストレージ (NVSラッパー)**:
    *   ESP32の不揮発性ストレージ（NVS）へのアクセス機能を、シンプルなTclコマンドである `nvs_set` および `nvs_get` にパッケージ化しました。
    *   再起動後も、接続先のサーバー構成（リモートサーバー、ポート、事前共有キー）を保持できます。
*   **セキュアネットワークトンネル (BiTun移植)**:
    *   セキュアなUDP/KCP SOCKS5プロキシトンネルであるBiTunを移植し、ESP32/FreeRTOS向けの堅牢なOS抽象化レイヤー（OSAL）を実装しました。
    *   暗号化と検証には **ChaCha20-Poly1305 AEAD** を使用し、インプレース（原位）復号と署名チェックを行うことで、強力なパケットロス耐性と改ざん防止を実現します。
    *   グローバルなシングルスレッド非同期DNSリゾルバとeventfd駆動型の起動メカニズムにより、SRAM使用量とスケジューラのオーバーヘッドを最適化しています。
*   **Tclによる動的ライフサイクル管理**:
    *   SOCKS5プロキシの実行状態を、Tclコンソールコマンド `bitun_start` および `bitun_stop` でバックグラウンドから動的に制御できます。
    *   起動時に最新のサーバー設定とキーをNVSから直接取得するため、ファームウェアの再コンパイルや再書き込みを行うことなく、リモート接続先を動的に変更できます。

## 3. クイックスタート (Quick Start)

### 3.1 前提条件
*   ESP-IDF v5.0以上 (推奨：**ESP-IDF v5.3**)。
*   CMake および Ninja ビルドツール。
*   Python 3.x。

### 3.2 ビルドと書き込み
TclスクリプトをCのバイト配列に自動変換し、ESP-IDFプロジェクトをビルドするためのシェルスクリプト `build.sh` を提供しています：

```bash
# 実行権限を付与
chmod +x build.sh

# ビルドスクリプトを実行
./build.sh
```

ビルドが完了したら、ファームウェアを書き込み、コンソールモニターを起動します（例：ESP32-C3チップ）：
```bash
idf.py -p <シリアルポート名> flash monitor
```

## 4. Tclコマンドリファレンス (Tcl Commands Manual)

対話型の Tcl Shell（シリアル端末または内蔵WebコンソールWebSocketからアクセス可能）に入ると、以下のカスタムコマンドを使用できます：

### 4.1 ハードウェアおよびシステム制御
*   **`gpio_mode <pin> <mode>`**: ピンモードを設定します。
    *   `mode`: `0` = 入力, `1` = 出力, `2` = 内部プルアップ抵抗付き入力。
*   **`digital_write <pin> <level>`**: 出力ピンに論理レベルを書き込みます。`level` は `0` または `1`。
*   **`digital_read <pin>`**: ピンの現在の論理状態を読み取ります。`0` または `1` を返します。
*   **`sleep <ms>`**: 指定されたミリ秒間、タスクの実行を一時停止します（内部的にFreeRTOSタスクをイールドします）。
*   **`log <on|off>`**: バックグラウンドのWi-Fiおよびシステムログの表示/非表示を切り替え、Tclコンソールの可読性を維持します。

### 4.2 ネットワーク診断
*   **`ipconfig`**: 現在のWi-Fi接続ステータス、ローカルIPアドレス、サブネットマスク、およびゲートウェイを表示します。
*   **`ping <host>`**: 対象のIPまたはドメイン名にICMP Echo要求を送信し、往復時間（RTT）の結果を表示します。

### 4.3 不揮発性ストレージ (NVS)
*   **`nvs_set <key> <value>`**: NVSに文字列のキーと値のペアを保存します。
    *   *注意*: `key` の長さは最大 15 バイトです。
*   **`nvs_get <key> ?default?`**: NVSから文字列を取得します。
    *   指定したキーが存在しない場合、オプショナル引数 `default` を返します（省略した場合は空文字列 `""`）。

### 4.4 BiTunトンネルライフサイクル制御
*   **`bitun_start`**: バックグラウンドで非同期にBiTunプロキシトンネルを起動します。
    *   起動時にNVSの `"storage"` ネームスペースから以下の設定を動的に取得します（見つからない場合はデフォルト値を使用します）：
        *   `bitun_rem_ip`: リモートKCPサーバーのIPアドレスまたはドメイン名（デフォルト：`"127.0.0.1"`）。
        *   `bitun_rem_port`: リモートKCPサーバーのリスニングポート（デフォルト：`9999`）。
        *   `bitun_loc_port`: ローカルSOCKS5プロキシポート（デフォルト：`1080`）。
        *   `bitun_psk`: 32バイトの事前共有キー文字列（デフォルト：`"00000000000000000000000000000000"`）。
*   **`bitun_stop`**: バックグラウンドで実行中のBiTunトンネルを停止し、アクティブなすべてのネットワークソケットとタスク割り当てをクリーンアップします。

## 5. 使用例 (Usage Example)

リモートサーバーに接続するには、Tclコンソールで以下のコマンドを実行します：

```tcl
# 1. リモートサーバーの設定と事前共有キーをNVSに保存する
nvs_set bitun_rem_ip "your-server-domain.com"
nvs_set bitun_rem_port "8888"
nvs_set bitun_psk "abcdefghijklmnopqrstuvwxyz123456"

# 2. トンネルプロキシを起動する
bitun_start

# 3. これにより、ローカルネットワーク内のデバイスから、ESP32の IP:1080 をSOCKS5プロキシとして設定し、セキュアなデータ転送を行えます。

# 4. 必要に応じてトンネルを停止する
bitun_stop
```
